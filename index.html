Softwares de Versionamento

Sistema que tem por objetivo realizar o controle de versões de um código ou documento. É possível gerenciar as versões de
um código ou documento realizadas por um desenvolvedor ou equipe de desenvolvedores.

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Software de Controle de Versões

Há o Repositório Local, onde voce pode guardar as versões do seu arquivo no PC. Enquanto que com o Repositório Remoto,
você pode guardar em um software de "nuvem". Exemplo de Remoto:

• GitHub

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Tipos de Softwares de Versão:

  • Centralizado: Possui um repositório central em um servidor com controles de versões.(Depende do servidor)
    Exemplo de software:


    - Subversion

  • Distribuído: Possui um repositório local com controle de versões. Atualizações são efetuadas no repositórrio central.(Independente do servidor)
    Exemplos de software:


    • - Mercurial


    - Git

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Git

• É um software livre de versionamento que surgiu para o desenvolvimento do Kernel do linux, foi projetado e desenvolvido por Linus Torvalds em 2005.
Cada diretório de trabalho do Git é um repositório com um histórico completo e habilidade total de acompanhamento das revisões , não dependentede acesso a uma rede ou a um servidor central.

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

GitHub

• É uma plataforma de hospedagem de código-fonte e arquivos com controle de versão. Surgiu em 2008 e foi adquirido em 2018 pelo Microsoft por US$ 7,5 bilhões.
• Mais ou menos 100 milhões de projetos;
• E mais de 56 milhões de usuários.

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Comandos Git:

• Comit:
Torna permanente os conjuntos de alterações em um repositório.
• Push:
Envia alterações para o servidor no repositório remoto.
• Fetch:
Verifica de o seu repositório local está em sincronia com o remoto.
• Pull:
Sincroniza o repositório local com o remoto, atualizando as versões.
